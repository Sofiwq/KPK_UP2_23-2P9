# Вариант S1: Load Assignment Service (Сервис распределения нагрузки)

## Функционал сервиса
Сервис предназначен для закрепления конкретного преподавателя за дисциплиной в конкретной группе.  
Пример: **«Иванов И.И. ведет Математику в группе 101»**.

### Основные сущности
- **Teacher** — преподаватель.
- **Group** — учебная группа.
- **Discipline** — дисциплина.
- **Assignment** — назначение, транзитивная сущность для связи Teacher, Group и Discipline.

### Правила валидации
- Все обязательные поля должны быть заполнены.
- Строковые поля не должны быть пустыми.
- Длина строк ограничена.
- Удаление реализуется логически через поле `is_active`.
- Внешние ключи не принимают `NULL`.
- Связь многие-ко-многим реализована через транзитивную таблицу `Assignment`.

## ER-диаграмма

![ER-диаграмма сервиса распределения нагрузки](erd.png)

Диаграмма показывает:
- `Teacher`, `Group`, `Discipline` — основные сущности.
- `Assignment` — транзитивная сущность.
- Связи:
  - `Teacher (1) -> (M) Assignment`
  - `Group (1) -> (M) Assignment`
  - `Discipline (1) -> (M) Assignment`

База данных соответствует **3НФ**, так как:
- каждая таблица описывает одну сущность;
- все неключевые поля зависят только от первичного ключа;
- транзитивные зависимости между неключевыми атрибутами отсутствуют;
- связь многие-ко-многим реализована через отдельную таблицу `Assignment`.
